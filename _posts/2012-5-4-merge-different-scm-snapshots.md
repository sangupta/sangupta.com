    This post is about MergeRepo a small script that allows to merge two different snapshots of the same repository from different SCMs into one.
    <br>
    <br>Recently, I had to collaborate on a project where the code was in Subversion. As we usually use Perforce, we forked a copy of the SVN trunk into Perforce and started our development. Once, the development was complete, we wanted to merge back in Subversion. And now, the nightmare started. SVN showed all files (thousands) to have been modified, due to a change in the timestamp or the change in line-ending character.
    <br>
    <br>The issue with line-ending character was easier to resolve, but the change in timestamp still created havoc. Thus, the need for this tool emerged. I wrote this simple script to ignore timestamp, and 
    <tt>EOL</tt> character, and merge the removed/added/modified files back into the SVN folder, so that only such files were shown for commit.
    <br>
    <br>The tool allows us to resolve such situations. It compares the two snapshots (folders/trees) and generates the merged repository with the only the modified files, so that it can be checked back into the original line. The tool can be used as,
    <br>
    <br>
    <pre class="brush: text">Usage:    $ java com.sangupta.keepwalking.MergeRepo &lt;previous&gt; &lt;newer&gt; &lt;destination&gt;<br><br>          previous      The folder corresponding to the older repository snapshot.<br>          newer         The folder corresponding to the newer repository snapshot.<br>          destination   The folder where the updated repository will be created.<br></pre>
